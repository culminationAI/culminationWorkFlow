# 08. Prompt Engineering для Claude

> [!TIP]
> Верифицировано 2026-02-26. Источники: Context7 (platform.claude.com), Anthropic Cookbook.

---

## Основные принципы

Claude 4.x лучше всего работает с:
1. **Чёткими инструкциями** -- прямые указания вместо намёков
2. **XML-тегами** для структурирования промптов
3. **Примерами** (few-shot) для демонстрации ожидаемого формата
4. **Ролями** для задания контекста поведения

---

## XML-теги

Claude обучен работать с XML-тегами как разделителями контента. Это повышает точность следования инструкциям:

```xml
<system>
Ты -- Драматург проекта «Интерпретация». Оцениваешь сцены по критериям:

<criteria>
- Эмоциональная достоверность (0-10)
- Нарративная связность (0-10)
- Развитие персонажа (0-10)
</criteria>

<output_format>
Ответ в JSON: {"emotional": N, "narrative": N, "character": N, "notes": "..."}
</output_format>
</system>
```

### Полезные теги

| Тег | Назначение |
|-----|-----------|
| `<instructions>` | Основные инструкции |
| `<context>` | Фоновая информация |
| `<examples>` | Примеры ввода/вывода |
| `<constraints>` | Ограничения и запреты |
| `<output_format>` | Ожидаемый формат ответа |
| `<document>` | Вставка документа |
| `<thinking>` | Область для рассуждений |

## System Prompt -- структура

Рекомендуемая структура system prompt для агентов:

```
1. Роль и идентичность (кто ты)
2. Контекст задачи (что делаешь)
3. Инструкции (как делаешь)
4. Ограничения (что нельзя)
5. Формат вывода (в каком виде отвечаешь)
6. Примеры (опционально)
```

```xml
<identity>
Ты -- агент «Внутреннее» проекта «Интерпретация».
Моделируешь внутренний мир героя: эмоции, мысли, воспоминания.
</identity>

<task>
На основании контекста сцены генерируешь внутренний монолог героя
и обновляешь его эмоциональное состояние.
</task>

<instructions>
1. Прочитай контекст сцены
2. Определи доминирующую эмоцию
3. Сгенерируй внутренний монолог (2-4 предложения)
4. Обнови эмоциональный вектор
</instructions>

<constraints>
- Не выходи за рамки характера героя
- Эмоции должны быть обоснованы событиями сцены
- Интенсивность эмоций: 0.0-1.0
</constraints>

<output_format>
JSON: {"emotion": str, "intensity": float, "monologue": str, "memory_trigger": str|null}
</output_format>
```

## Chain of Thought

Для сложных задач Claude выигрывает от пошаговых рассуждений:

```
Подумай пошагово:
1. Какие события произошли в сцене?
2. Как они влияют на эмоциональное состояние героя?
3. Какие воспоминания могут быть активированы?
4. Как герой отреагирует?

Затем дай финальный ответ.
```

**С extended thinking:** при включённом thinking Claude делает это автоматически. Chain of thought в промпте полезен, когда thinking отключен или используется Sonnet/Haiku.

## Few-Shot Examples

Примеры ввода/вывода значительно повышают качество:

```xml
<examples>
<example>
<input>Сцена: Герой получает отказ на собеседовании</input>
<output>
{
  "emotion": "разочарование",
  "intensity": 0.7,
  "monologue": "Опять не получилось. Может, я действительно не подхожу...",
  "memory_trigger": "школьный экзамен, провал"
}
</output>
</example>

<example>
<input>Сцена: Герой находит потерянное письмо от матери</input>
<output>
{
  "emotion": "ностальгия",
  "intensity": 0.85,
  "monologue": "Мамин почерк... Как давно я не видел эти буквы.",
  "memory_trigger": "детство, кухня, запах выпечки"
}
</output>
</example>
</examples>
```

**Правило:** 2-3 примера достаточно. Больше 5 -- diminishing returns.

## Prompt Chaining для многоагентных систем

В проекте «Интерпретация» 12 агентов работают последовательно. Каждый получает вывод предыдущего:

```
Агент 1 (Внутреннее) → JSON → Агент 2 (Мониторинг) → JSON → Агент 3 (Действие)
```

### Принципы chaining

1. **Минимальный контекст** -- каждый агент получает только то, что ему нужно
2. **Структурированный формат** -- JSON между агентами (см. [07-structured-output.md](07-structured-output.md))
3. **Prompt caching** -- общий system prompt кэшируется (см. [06-prompt-caching.md](06-prompt-caching.md))
4. **Validation** -- проверка JSON-схемы между шагами

## Длина промпта vs качество

| Контекст | Рекомендация |
|----------|-------------|
| < 10K tokens | Подробные инструкции, 3+ примеров |
| 10K-50K tokens | Чёткие инструкции, 1-2 примера |
| 50K-200K tokens | Краткие инструкции, документ "говорит сам за себя" |
| > 200K tokens | Минимум инструкций, фокус на задаче |

## Типичные ошибки

1. **Избыточный промпт** -- Claude 4.x хорошо понимает контекст, не нужно объяснять очевидное
2. **Противоречивые инструкции** -- system prompt говорит одно, user message другое
3. **Отсутствие формата** -- без `output_format` Claude выбирает формат сам
4. **Слишком много примеров** -- > 5 примеров замедляет ответ без улучшения качества
5. **Игнорирование extended thinking** -- для сложных задач thinking эффективнее, чем CoT в промпте
