## Обзор

Claude может анализировать изображения и PDF-документы, переданные через Messages API. Это позволяет:

- Описывать содержание изображений
- Извлекать текст (OCR)
- Анализировать графики, диаграммы, схемы
- Сравнивать несколько изображений
- Обрабатывать PDF-документы

---

## Поддерживаемые форматы

### Изображения

| Формат | MIME-тип |
|--------|----------|
| JPEG | `image/jpeg` |
| PNG | `image/png` |
| GIF | `image/gif` |
| WebP | `image/webp` |

### Ограничения

- Максимальный размер файла: 20 МБ на изображение
- Максимальное разрешение: зависит от формата, рекомендуется не более 8000x8000
- До 20 изображений в одном запросе (зависит от контекстного окна)

### PDF

- Поддерживается как документ (каждая страница конвертируется в изображение)
- Текстовые PDF также обрабатываются через визуальное представление

## Передача изображений

### Base64

```json
{
  "role": "user",
  "content": [
    {
      "type": "image",
      "source": {
        "type": "base64",
        "media_type": "image/jpeg",
        "data": "/9j/4AAQSkZJRg..."
      }
    },
    {
      "type": "text",
      "text": "Опиши, что изображено на фотографии."
    }
  ]
}
```

### URL

```json
{
  "role": "user",
  "content": [
    {
      "type": "image",
      "source": {
        "type": "url",
        "url": "https://example.com/photo.jpg"
      }
    },
    {
      "type": "text",
      "text": "Что на этом изображении?"
    }
  ]
}
```

**Примечание:** URL-метод требует, чтобы изображение было доступно по прямой ссылке. Для приватных файлов используйте base64.

## Подсчет токенов для изображений

Стоимость изображений в токенах зависит от разрешения. Примерная формула:

- Изображение масштабируется так, чтобы вписаться в максимальный размер
- Затем разбивается на тайлы (tiles)
- Каждый тайл стоит определенное количество токенов

**Приблизительные значения:**

| Разрешение | Примерная стоимость |
|-----------|-------------------|
| 100x100 (мелкое) | ~50-100 tokens |
| 500x500 (среднее) | ~300-500 tokens |
| 1000x1000 (большое) | ~800-1600 tokens |
| 2000x2000 (очень большое) | ~2000-3500 tokens |

**Точная формула:** зависит от реализации и может меняться. Рекомендуется проверять через поле `usage.input_tokens` в ответе.

## Пример на Python

### Одно изображение

```python
import anthropic
import base64
from pathlib import Path

client = anthropic.Anthropic()

# Чтение и кодирование изображения
image_data = base64.standard_b64encode(
    Path("photo.jpg").read_bytes()
).decode("utf-8")

message = client.messages.create(
    model="claude-opus-4-6",
    max_tokens=1024,
    messages=[
        {
            "role": "user",
            "content": [
                {
                    "type": "image",
                    "source": {
                        "type": "base64",
                        "media_type": "image/jpeg",
                        "data": image_data,
                    },
                },
                {
                    "type": "text",
                    "text": "Опиши обстановку на этом фото. Что за место, какое время суток, какая атмосфера?"
                },
            ],
        }
    ],
)

print(message.content[0].text)
```

### Несколько изображений

### PDF-документ

```python
import base64
from pathlib import Path

pdf_data = base64.standard_b64encode(
    Path("document.pdf").read_bytes()
).decode("utf-8")

message = client.messages.create(
    model="claude-opus-4-6",
    max_tokens=4096,
    messages=[
        {
            "role": "user",
            "content": [
                {
                    "type": "document",
                    "source": {
                        "type": "base64",
                        "media_type": "application/pdf",
                        "data": pdf_data,
                    },
                },
                {
                    "type": "text",
                    "text": "Извлеки ключевые тезисы из этого документа."
                },
            ],
        }
    ],
)
```

## Best Practices

1. **Размещайте изображение перед текстом.** Claude лучше обрабатывает изображения, если они идут первыми в массиве content.

2. **Оптимизируйте разрешение.** Не отправляйте 4K-фотографии, если достаточно 1000x1000. Меньшее разрешение = меньше токенов = быстрее и дешевле.

3. **Конкретные вопросы.** Вместо "Что на фото?" спрашивайте конкретно: "Сколько людей на фото? Какое время суток? Что за место?"

4. **Несколько изображений.** Claude может обрабатывать до 20 изображений за раз. Полезно для сравнений и анализа серий.

5. **PDF.** Для текстовых PDF рассмотрите извлечение текста заранее -- это дешевле, чем передача PDF как изображений. Для PDF с графиками и таблицами -- передавайте как документ.

## Ограничения Vision

- **Нет генерации изображений.** Claude анализирует изображения, но не создает их.
- **Мелкий текст.** Очень мелкий текст на изображениях может распознаваться с ошибками.
- **Пространственное позиционирование.** Точные координаты объектов на изображении -- приблизительные.
- **Подсчет объектов.** При большом количестве объектов подсчет может быть неточным.

---

## Применение в проекте «Интерпретация»

Vision может использоваться на этапе **создания героя** (Фаза 0, бриф):

- Анализ фотографий реальных людей для создания прообразов NPC
- Анализ фотографий мест для описания локаций
- Разбор PDF с бэкстори или брифом

В самой симуляции vision не используется -- все данные хранятся в графе как текст.
