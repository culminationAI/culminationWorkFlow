# 11. Контекстное окно и оптимизация расхода токенов

> [!TIP]
> Верифицировано 2026-02-26. Источники: support.claude.com (статьи 11647753, 8606394, 8606395, 9797557, 12429409), platform.claude.com (Context Windows, Compaction), anthropic.com (Effective Context Engineering).

---

## Два типа лимитов: Usage vs Length

Это **разные** ограничения, и путать их -- главная ошибка при работе с Claude.

| Тип | Что ограничивает | Сброс | Как обойти |
|-----|------------------|-------|------------|
| **Usage limits** | Объём использования за период (session + weekly) | Session: каждые 5 часов. Weekly: еженедельно | Экономить токены, выбирать модель, extra usage |
| **Length limits** | Размер контекстного окна (одна беседа) | Не сбрасывается -- начните новый чат | Compaction, новый чат, Projects с RAG |

> [!IMPORTANT]
> Использование claude.ai, Claude Code и Claude Desktop считается в **один и тот же** лимит. Если вы активно используете Claude Code -- это съедает тот же бюджет, что и чат на claude.ai.

## Контекстное окно: размеры по планам

| Платформа | Стандартный контекст | Расширенный контекст |
|-----------|---------------------|---------------------|
| claude.ai (Pro, Max) | 200K tokens | — |
| claude.ai (Enterprise) | 200K tokens (500K для Sonnet 4.5) | — |
| API (Opus 4.6, Sonnet 4.6, Sonnet 4.5, Sonnet 4) | 200K tokens | до 1M tokens (beta, tier 4+) |

### Как считается контекст

Контекстное окно = **вся** информация, которую модель видит при генерации ответа:

```
context_window = system_prompt
              + tool_definitions
              + conversation_history (все предыдущие turn'ы)
              + текущее сообщение пользователя
              + ответ модели (включая thinking)
```

**Линейный рост:** каждый следующий turn включает ВСЕ предыдущие. 10 обменов по 2K токенов = уже ~20K+ в контексте на 10-м turn'е.

## Что съедает токены (от самого дорогого к дешевому)

### 1. Extended Thinking (Расширенное мышление)

Extended thinking -- главный потребитель токенов. Thinking-блоки могут генерировать **десятки тысяч** output-токенов за один ответ.

| Аспект | Детали |
|--------|--------|
| Стоимость | Тарифицируется как output ($25/1M для Opus 4.6) |
| Влияние на контекст | Thinking-блоки **автоматически удаляются** из контекста последующих turn'ов |
| Влияние на usage | Thinking-токены **полностью** учитываются в usage limits |

**Формула контекста с thinking:**

```
effective_context = (input_tokens - previous_thinking_tokens) + current_turn_tokens
```

> [!CAUTION]
> При подписке Max x20 extended thinking -- это **главная причина** быстрого исчерпания лимитов. Thinking budget в 100K+ токенов на один запрос может выжечь 5-10% сессионного лимита за одно сообщение.

### 2. Tools и Connectors (инструменты)

Определения инструментов занимают значительное количество токенов и присутствуют **в каждом** запросе:

- Каждый tool definition: ~200-1000 токенов (включая описание и JSON schema)
- Web search, Research, MCP connectors: добавляют токены в каждый запрос
- Tool results: каждый результат вызова инструмента занимает контекст

### 3. Файлы и изображения в сообщениях

| Тип | Примерный расход |
|-----|-----------------|
| Текст | 1 токен ≈ 4 символа EN / ≈ 1-2 символа RU |
| Изображения | 1 500-170 000 токенов (зависит от размера, пересчитывается в tiles) |
| PDF | Поcтраничная обработка, каждая страница как изображение |

### 4. Длинная история беседы

Каждый turn полностью сохраняется. В длинном чате с 50 обменами контекст заполняется **историей**, а не полезным контентом.

## Оптимизация для Max x20: практические стратегии

### Уровень 1: Немедленная экономия (30-50%)

#### Отключайте Extended Thinking, когда он не нужен

Extended thinking необходим для:
- Сложного многоступенчатого рассуждения
- Написания длинного кода
- Анализа больших документов

Extended thinking **не нужен** для:
- Простых вопросов и ответов
- Редактирования текста
- Классификации и фильтрации
- Перевода
- Форматирования

**Экономия:** 10K-200K output-токенов на каждый запрос, где thinking отключён.

#### Отключайте неиспользуемые инструменты

В Settings > Search and tools отключите:
- Web search -- если не нужен поиск в интернете
- Research -- если не ведёте исследование
- MCP connectors -- если не используются

**Экономия:** 2K-10K input-токенов на каждый запрос.

#### Начинайте новый чат для новых тем

Каждый новый чат начинается с пустого контекста. Если вы переходите к новой задаче -- новый чат экономит все токены, которые ушли бы на перенос истории.

### Уровень 2: Структурная оптимизация (20-40% дополнительно)

#### Используйте Projects с RAG

Projects в claude.ai используют **Retrieval Augmented Generation**:
- Документы загружаются один раз и **кэшируются**
- В контекст загружается только **релевантная часть** документа, а не весь файл
- Повторные обращения к тем же документам расходуют значительно меньше токенов

**Как использовать:**
1. Создайте Project в claude.ai
2. Загрузите рабочие документы в Project Knowledge
3. Пишите краткие, сфокусированные Project Instructions (не дублируйте содержание файлов)
4. Регулярно удаляйте ненужные файлы из проекта

#### Батчинг запросов

Вместо 5 отдельных сообщений с вопросами -- одно сообщение с 5 вопросами:

```
❌ Плохо (5 отдельных запросов = 5x контекст):
1. "Как сделать X?"
2. "А как Y?"
3. "А ещё Z?"
4. "А что насчет W?"
5. "И V?"

✅ Хорошо (1 запрос):
"Ответь на 5 вопросов:
1. Как сделать X?
2. Как Y?
3. Как Z?
4. Что насчет W?
5. И V?"
```

**Экономия:** system prompt + tool definitions передаются только 1 раз вместо 5.

#### Будьте конкретны в промптах

```
❌ Плохо: "Расскажи всё о машинном обучении"
✅ Хорошо: "Объясни разницу между L1 и L2 регуляризацией, 3-4 предложения"
```

Чем конкретнее промпт, тем короче ответ, тем меньше output-токенов.

### Уровень 3: Продвинутая оптимизация (API)

#### Prompt Caching (подробнее в [06-prompt-caching.md](06-prompt-caching.md))

При использовании API -- кэшируйте всё, что повторяется:

| Что кэшировать | TTL | Экономия |
|----------------|-----|----------|
| System prompt | `"1h"` | до 90% на input |
| Tool definitions | `"1h"` | до 90% на input |
| Документы-контексты | `"5m"` | до 90% на input |

Cache read стоит **$0.50/1M** vs **$5.00/1M** за обычный input (Opus 4.6).

#### Compaction (beta, только Opus 4.6)

Server-side compaction автоматически сжимает ранние части разговора:

```python
# API: включение compaction
response = client.messages.create(
    model="claude-opus-4-6",
    max_tokens=4096,
    # compaction -- серверная функция, автоматически
    # активируется при приближении к лимиту контекста
    messages=[...]
)
```

В claude.ai compaction работает **автоматически** при включённом Code Execution:
- Claude суммаризирует ранние сообщения
- Полная история сохраняется для справки
- Вы увидите "Claude is organizing its thoughts"

#### Context Awareness (Sonnet 4.6+, Haiku 4.5+)

Модели Sonnet 4.6, Sonnet 4.5 и Haiku 4.5 **знают** свой оставшийся бюджет токенов:

```xml
<!-- Модель видит при инициализации: -->
<budget:token_budget>200000</budget:token_budget>

<!-- После каждого tool call: -->
<system_warning>Token usage: 35000/200000; 165000 remaining</system_warning>
```

Это позволяет модели планировать длинные задачи и не обрубаться посередине.

> [!NOTE]
> Opus 4.6 НЕ имеет встроенной контекстной осведомлённости (по состоянию на февраль 2026). Эта функция доступна в Sonnet 4.6, Sonnet 4.5 и Haiku 4.5.

#### Выбор модели под задачу

| Задача | Модель | Почему |
|--------|--------|--------|
| Сложное рассуждение, агенты | Opus 4.6 | Максимальная глубина |
| Повседневные задачи | Sonnet 4.6 / 4.5 | Дешевле, context awareness |
| Классификация, фильтрация | Haiku 4.5 | В 6x дешевле Opus на input |

**Один запрос к Opus 4.6 с thinking ≈ 5-20 запросов к Haiku 4.5 по стоимости.**

## Мониторинг расхода

### В claude.ai

Settings > Usage показывает:
- **Current session** -- прогресс текущей 5-часовой сессии
- **Weekly limits** -- отдельно для Opus и all other models
- **Extra usage** -- потраченные средства сверх лимита (если включен)

### Opus -- отдельный weekly лимит

> [!WARNING]
> Opus имеет **отдельный** недельный лимит от остальных моделей. 27% за полдня = примерно 54% за полный день = лимит может закончиться за ~2 дня интенсивной работы.

### Стратегия распределения лимита Max x20

| День недели | Рекомендуемый расход Opus | Стратегия |
|-------------|--------------------------|-----------|
| Пн-Ср | 10-15% / день | Основная работа, thinking включён по необходимости |
| Чт-Пт | 10-15% / день | Основная работа |
| Сб-Вс | 5-10% / день | Резерв или дополнение |
| **Буфер** | ~10-15% | На непредвиденные задачи |

### Самые «дорогие» сценарии использования

1. **Длинный чат + Extended Thinking + множество инструментов** -- максимальный расход
2. **Research mode** -- множественные поисковые запросы, быстрое потребление токенов
3. **Claude Code** -- интенсивное использование tools, каждая итерация -- полный цикл контекста
4. **Загрузка больших файлов/изображений в каждое сообщение** -- без проектов нет кэширования

## Automatic Context Management (claude.ai)

При включённом Code Execution в claude.ai работает **автоматическое управление контекстом:**

1. Когда разговор приближается к лимиту контекстного окна (200K), Claude суммаризирует ранние сообщения
2. Полная история сохраняется и доступна для справки
3. Суммаризация **не считается** в usage limits
4. Позволяет вести практически бесконечные разговоры

**Ограничения:**
- Требуется включённый Code Execution
- Очень большие первые сообщения могут создать проблемы
- Суммаризация может потерять детали ранних сообщений

## Extra Usage (дополнительное использование)

Если лимиты Max x20 исчерпаны, можно включить Extra Usage:

| Параметр | Значение |
|----------|----------|
| Тарификация | По стандартным API-ценам |
| Активация | Settings > Usage > Enable Extra Usage |
| Контроль | Monthly spending cap + auto-reload |
| Лимит пополнения | $2000/день |

**Расчёт стоимости:** при использовании Opus 4.6 с Extended Thinking, один длинный разговор (50 обменов) может стоить $5-50 в extra usage в зависимости от объёма thinking.

## Чек-лист: ежедневная оптимизация

- [ ] **Перед началом:** проверьте текущий расход в Settings > Usage
- [ ] **Модель:** выбирайте Opus только для сложных задач, Sonnet/Haiku для остальных
- [ ] **Thinking:** отключайте для простых задач
- [ ] **Инструменты:** отключайте неиспользуемые (web search, research, MCP)
- [ ] **Новый чат:** каждая новая тема = новый чат
- [ ] **Батчинг:** группируйте связанные вопросы в одно сообщение
- [ ] **Промпты:** будьте конкретны, указывайте желаемый объём ответа
- [ ] **Projects:** используйте для документов, к которым обращаетесь многократно
- [ ] **Мониторинг:** отслеживайте расход после каждой интенсивной сессии

## Формула оценки расхода

Примерная формула для оценки «стоимости» одного сообщения в терминах лимита:

```
message_cost ∝ input_tokens + output_tokens + thinking_tokens

Где:
  input_tokens  = system_prompt + tools + history + current_message
  output_tokens = длина ответа
  thinking_tokens = объём extended thinking (0, если выключено)
```

**Пример:** чат с 10 предыдущими обменами (по ~2K на обмен), thinking на 50K:

```
input ≈ 5K (system) + 3K (tools) + 20K (history) + 1K (message) = 29K input
output ≈ 2K text + 50K thinking = 52K output
total ≈ 81K tokens за ОДНО сообщение
```

При Max x20 это может составить ~1-3% сессионного лимита.
