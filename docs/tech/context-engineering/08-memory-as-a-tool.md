# Memory-as-a-Tool: Стратегии инференса

Память не должна быть пассивным грузом. В продвинутых системах агент сам управляет тем, когда и как ему «вспоминать».

## 1. Память как инструмент (Self-Triggered Retrieval)
Вместо того чтобы автоматически добавлять память в каждый промпт, мы даем агенту инструмент `search_memory`.
- **Зачем**: Это экономит токены. Если мы обсуждаем ядерную физику, агенту не нужно помнить, что пользователь любит пиццу.
- **Decision Engine**: LLM анализирует запрос и решает: "Мне нужно заглянуть в историю, чтобы ответить на это".

## 2. Типы извлечения (Retrieval Strategies)
- **Top-K Vector Retrieval**: Берем 5 самых «похожих» фрагментов из Qdrant.
- **Sub-graph Extraction**: Берем узел "Иван" в Neo4j и все связанные с ним узлы в радиусе 2-х прыжков.
- **Temporal Slicing**: Берем только события, произошедшие за последние 3 игровых дня.

## 3. Инференс с памятью (Inference with Memory)
Как вставить память в промпт?
1.  **System Prompt injection**: Постоянные факты ("Ты — детектив из 40-х") всегда там.
2.  **Context augmentation**: Динамические факты ("У тебя в кармане ключ от сейфа") добавляются только когда это уместно.
3.  **Few-shot selection**: Выбор примеров из памяти, которые максимально похожи на текущую ситуацию.

## 4. Рефлексия во время инференса (Reflection)
Перед тем как ответить, агент может провести "внутренний диалог":
- "Я собираюсь сказать А, но помню ли я Б?"
- "Не противоречит ли мой ответ факту из прошлого?"
Это значительно снижает уровень галлюцинаций.
