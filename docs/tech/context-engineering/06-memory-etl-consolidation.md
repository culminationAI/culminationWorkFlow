# Memory ETL: Консолидация и Происхождение (Provenance)

Когда фактов становится много, они начинают противоречить друг другу. Консолидация — это процесс превращения разрозненных заметок в единую, непротиворечивую картину мира.

## 1. Консолидация (Consolidation)
Это «сон» агента, во время которого он наводит порядок.
- **Deduplication**: Если из пяти сессий извлечено "Имя: Иван", в памяти должен остаться один узел с этим именем.
- **Conflict Resolution**: Что если вчера пользователь сказал "Я живу в Лондоне", а сегодня "Я переехал в Париж"? 
    - *Решение*: Использование временных меток (Timestamps). Новая информация имеет приоритет, но старая сохраняется в истории.
- **Generalization**: Превращение частных случаев в правила. "Пользователю не нравится, когда я шучу про погоду" -> Правило стиля общения.

## 2. Происхождение (Provenance/Lineage)
В "Ultimate" системе каждое воспоминание должно иметь «чек об оплате». Мы должны знать, откуда взялся этот факт.
- **Session ID**: Из какой беседы это пришло?
- **Timestamp**: Когда это было сказано?
- **Confidence**: Насколько мы уверены?
- **Source Verification**: Был ли этот факт подтвержден пользователем или это предположение агента?

## 3. Очистка (Memory Pruning)
Не всё нужно помнить десятилетиями.
- **Decay Function**: У каждой записи есть "пульс". Если факт не запрашивался год, его можно переместить в архив или удалить.
- **Correction**: Возможность для пользователя сказать: "Нет, ты запомнил неправильно". Система должна уметь «удалять» связи в графе.
