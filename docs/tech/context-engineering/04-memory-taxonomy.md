# Memory Taxonomy: Виды и виды памяти

Память агента в концепции Google — это не «одна большая папка». Это сложная иерархия хранилищ, каждое из которых отвечает за свой аспект «сознания».

## 1. Пять уровней памяти
Согласно whitepaper, "Ultimate" агент должен обладать следующими типами памяти:

### A. Episodic Memory (Эпизодическая)
- **Что это**: Пересказ прошлых бесед и событий жизни.
- **Зачем**: "Мы вчера обсуждали этот план".
- **Формат**: Текстовые резюме сессий.

### B. Semantic Memory (Семантическая)
- **Что это**: База знаний о мире.
- **Зачем**: Отраслевые факты, правила физики, лор игры.
- **Формат**: Векторная БД (Qdrant).

### C. Fact Memory (Фактическая/Декларативная)
- **Что это**: Четкие, дискретные данные.
- **Зачем**: "Имя: Иван", "Телефон: +123...", "Любимый цвет: Зеленый".
- **Формат**: Граф знаний (Neo4j) или структурированный JSON.

### D. Procedural Memory (Процедурная)
- **Что это**: Знание агентом того, «как» что-то делать.
- **Зачем**: Инструкции по использованию инструментов, алгоритмы действий.
- **Формат**: System prompt, Tool definitions.

### E. Experiential Memory (Эмпирическая/Опытная)
- **Что это**: Память о собственных успехах и ошибках.
- **Зачем**: "В прошлый раз, когда я предложил это, пользователь рассердился. Больше так не делаю".
- **Формат**: Рефлексии и "Learned Rules".

## 2. Организационные паттерны
Как информация распределяется в памяти:
- **Centralized**: Одна БД на всех агентов. (Легко управлять, но сложно разделять права).
- **Decentralized**: У каждого агента своя память. (Высокая приватность, но сложно синхронизировать).
- **Hybrid**: Наша цель. Общий граф знаний о мире + личные "дневники" (сессии) агентов.

## 3. Масштаб и Жизненный цикл (Memory Scope)
- **Volatile**: Исчезает после сессии (Working Memory).
- **Persistent**: Живет вечно (Long-term DB).
- **Archival**: Сжатые, редко используемые данные в холодном хранилище.
